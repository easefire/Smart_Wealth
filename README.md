Smartwealth金融理财项目基于金融业务场景，业务逻辑主要包括申购，赎回，查看产品数据，产品净值更新、订单每日收益结算、每日对账等等。
项目采用模块化单体架构，分为用户模块，资产模块，交易模块，产品模块等，模块只能访问自己的数据库表，如果访问其他的模块需要通过其他模块的内部service来拿到数据，实现了数据解耦。
项目的主要亮点有以下几个，
首先，针对高频的申购操作场景，我采用Redis Lua脚本库存预扣减来防止超卖，然后采用本地消息表和消息队列来实现异步资产端的钱包扣款，确保消息的可靠性和数据一致性，在Jmeter的测试下用我的电脑能达到3000QPS。
第二，针对热点产品的高频访问场景，我根据数据的特性把它分为了静态数据（比如产品净值详情信息变动小）和动态数据（比如库存变动高频）针对静态数据采用了基于本地缓存和Redis和mysql的三级缓存设计，针对动态数据我设置了极短的本地缓存TTL和Redis进行动态回填组装数据返回，极大降低数据库压力
第三，针对持有中订单每日海量收益结算以及对账需求，我采用XXLJOB分片广播策略结合线程池将用户数据水平拆分到多个节点，来缩短任务执行时间同时多节点也提供了可靠性保障，并在任务完成以后进行缓存预热，校准MySQL与Redis库存的数据漂移问题
